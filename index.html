2 - Команды работы с ветками

Дополнительные команды

  1. git checkout NAME/HASH - позволяет перместитьна ветку или коммит ХЕД
  2. git checkout -b NEW_BRANCH_NAME - созвать новую ветку и сразу переместиться на нее
  3. git merge BRANCH_NAME - влить BRANCH_NAME в текущую ветку

Базовые команды

  • git init - создание репозитория
  • git status - показывает ваше текущее состояние в гите
  • git add - добавление файлов из рабочей директории в стейдж
  • git commit -m "commit description" - (с флагом М) создание коммита (сохранение изменений, накопленных в стейдже)
  • git push - отправить все новые коммиты на сервер (синхнонизация изменений с сервером)
  • git pull - забрать все новые коммиты с сервера (синхнонизация изменений с сервером) и переключится на самый свежий комит
  • git fetch - забрать все новые коммиты с сервера (синхнонизация изменений с сервером)

Три основных места работы:

1. WD (Working directory) - рабочая директория. Место где мы храним и изменяем файлы

2. Staging area (stage) - стейдж. Место, куда попадают файлы после команды git add. По сути, это промежуточный этап, в котором мы СОБИРАЕМ изменения для того чтобы затем на их основе создать коммит.

3. Local Repo - локальный репозиторий. Место хранения наших коммитов (измеений)

4. Remote repo - удаленный репозиторий. То же самое, но не на нашем компьютере, а где-то на отдельном сервере.
